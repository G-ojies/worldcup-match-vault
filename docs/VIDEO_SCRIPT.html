Part 1 — Pre-flight (before you hit record)

  1. Clean browser profile. Use a fresh profile with only your Solana wallet (Phantom/Solflare) enabled — disable every other wallet extension. This removes the window.cardano extension error that pops a red full-screen overlay.
  2. Wallet → Devnet, funded. Switch the wallet to Devnet and fund ~0.3 SOL from faucet.solana.com.
  3. URL: use production — https://worldcup-match-vault.vercel.app (it proves the build is deployed). Fallback: localhost:3000 via cd app && npm run dev.
  4. Open tabs, in order: (1) the dApp, (2) Solana Explorer set to devnet (explorer.solana.com/?cluster=devnet), (3) the GitHub repo.
  5. Recorder: OBS Studio (or SimpleScreenRecorder) — capture screen + microphone at 1080p. Record a 10-second test and play it back to check audio.
  6. Zoom the browser to ~110–125% for legibility. Close notifications, silence your phone.
Know this quirk: the Netherlands vs Morocco market's live-odds header may be blank — that TxLINE fixture aged out of the live feed. The pools and settlement still work perfectly. So: demo live odds on the England market, and use Netherlands vs Morocco only for bet → settle → claim.

And here's the receipt. The two proven stat leaves — home one, away one. The predicate: home minus away equals zero, so the outcome is Draw. Part 3 — Delivery tips

  • Lead with the differentiator. The settlement moment (1:55–3:20) is what wins — give it room, slow down, let the receipt breathe on screen.
  • Pause on approvals. When a wallet popup appears, stop narrating for a beat, approve, then resume.
  • Say the words "no oracle authority" at least twice — it's the core claim of the whole project.
  • Don't rush the receipt. Judges need ~4–5 seconds to read the proven leaves, predicate, and PDA.
  • If you stumble, keep going — you can trim in post. One clean take of the settle moment matters most.

Part 4 — After recording
